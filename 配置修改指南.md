# 配置修改指南

本文档面向非技术人员，说明如何修改系统的角色库、场景配置和坐标文件，无需接触任何代码。

所有需要修改的文件都是 `.json` 格式，可以用任意文本编辑器打开（推荐 VS Code 或记事本++）。修改时请注意保持格式正确，每个大括号、方括号、引号都要成对出现。

---

## 一、角色库 `characters_resource.json`

**文件位置：** `backend/resources/characters_resource.json`

这是系统已知的所有可用角色模型的档案。AI 生成剧本时会从这里查找角色信息，生成的演员档案文件也以这里的数据为基础。

### 每个角色的字段说明

```json
{
  "name": "TR-4 检修者",         ← 角色名称，AI 在剧本中按此名称识别
  "age": 15,                     ← 年龄（数字，机器人可填运行年限）
  "gender": "none",              ← 性别："male" / "female" / "none"
  "gameobject_name": "TR_worker",← Unity 场景中对应的对象名（必填，不能为空）
  "appearance": {
    "height": "195cm",           ← 身高
    "body_type": "...",          ← 体型与外观描述
    "hair": "...",               ← 头发/头部特征描述
    "face": "..."                ← 面部特征描述
  },
  "acting_style": "...",         ← 表演风格描述（动作习惯、走路方式等）
  "traits": [                    ← 性格标签，3个左右
    "执行力强",
    "功能至上"
  ],
  "background": "..."            ← 角色背景故事
}
```

### 新增角色

在文件末尾 `]` 之前，仿照上面的格式添加一个新的 `{...}` 对象，用逗号与前一个隔开：

```json
  },               ← 上一个角色的结尾
  {                ← 新角色开始
    "name": "新角色名称",
    "age": 0,
    "gender": "male",
    "gameobject_name": "对应Unity对象名",
    "appearance": {
      "height": "175cm",
      "body_type": "描述",
      "hair": "描述",
      "face": "描述"
    },
    "acting_style": "描述",
    "traits": ["标签1", "标签2"],
    "background": "背景故事"
  }
]
```

> **注意：** `gameobject_name` 必须与 Unity 工程中实际存在的对象名一致，否则角色无法在场景中加载。

### 修改已有角色

直接找到对应角色，修改对应字段的引号内容即可。名称（`name`）修改后，前端选人界面中显示的名字也会同步变化。

## 二、场景配置 `scenes_resource.json`

**文件位置：** `backend/resources/scenes_resource.json`

定义系统中所有可用的场景，包括场景的点位分布和镜头分组。

### 场景结构说明

```json
{
  "id": "Space Station",         ← 场景唯一标识（前端下拉菜单的选项值）
  "name": "太空站",              ← 场景显示名称
  "style_tag": "Cyberpunk_Realism", ← 风格标签（仅描述用）
  "description": "场景整体描述，AI 会参考这段描述来创作剧情",
  "camera_groups": [...],        ← 镜头分组（见下方说明）
  "valid_positions": [...]       ← 场景内所有可用点位（见下方说明）
}
```

### 点位（`valid_positions`）

每个点位代表角色在场景中可以站立的一个具体位置：

```json
{
  "id": "Position 1",            ← 点位编号，必须唯一
  "camera_group": "A",           ← 所属镜头组（必须与 camera_groups 中的 id 对应）
  "description": "中央指挥台，适合发号施令、战略决策",  ← AI 根据此描述选择合适点位
  "is_sittable": false           ← 该点位是否可以坐下：true / false
}
```

`description` 字段是 AI 选点位的唯一依据，写得越具体准确，AI 选位越合理。

### 镜头分组（`camera_groups`）

同一个对白片段中，所有角色必须属于同一个镜头组。分组代表摄影机能同时拍到的空间范围：

```json
{
  "id": "A",                     ← 分组字母，对应点位中的 camera_group 字段
  "name": "主控室",              ← 分组名称（描述用）
  "position_ids": ["Position 1", "Position 2", "Position 3"]  ← 该组包含的点位
}
```

> **重要：** 新增点位后，必须将其 `id` 加入对应 `camera_group` 的 `position_ids` 列表中，否则点位不会被镜头分组规则识别。

### 新增场景

在文件末尾 `]` 之前，仿照现有格式完整添加一个新的场景对象。`id` 字段必须全局唯一。

## 三、坐标文件 `positions_sample.json`

**文件位置：** `backend/resources/positions_sample.json`

存储每个场景中各点位的真实 3D 坐标，供 Unity 使用。前端"下载坐标文件"按钮下载的就是这个文件。

### 结构说明

```json
{
  "Space Station": {                          ← 场景 id（必须与 scenes_resource.json 中的 id 一致）
    "Position 1": {
      "position": [30, 0, -14],              ← Unity 坐标 [x, y, z]
      "sit_angle": [],                        ← 坐下时的朝向角度（通常留空）
      "fixed_angle": [0, -161.5, 0]          ← 站立时的朝向角度 [x旋转, y旋转, z旋转]
    },
    "Position 2": { ... }
  }
}
```

### 新增点位坐标

在对应场景下，按照同样格式添加新点位的坐标条目。点位名称（`"Position N"`）必须与 `scenes_resource.json` 中的 `id` 完全一致（大小写和空格都要匹配）。

## 常见操作速查

| 我想做的事 | 修改的文件 | 修改的字段 |
|-----------|-----------|-----------|
| 新增一个可用角色模型 | `characters_resource.json` | 新增完整角色对象 |
| 修改角色的背景描述 | `characters_resource.json` | `background` |
| 新增一个场景 | `scenes_resource.json` | 新增完整场景对象 |
| 新增场景中的站位点 | `scenes_resource.json` | `valid_positions` + `camera_groups` |
| 修改点位的空间描述 | `scenes_resource.json` | `valid_positions[].description` |
| 更新 Unity 坐标 | `positions_sample.json` | 对应点位的 `position` 和 `fixed_angle` |

---

## 修改后的注意事项

1. **保存前检查格式：** JSON 文件中所有字符串必须用英文双引号 `"` 包裹，每个字段之间用英文逗号 `,` 分隔，最后一个字段后面不加逗号。
2. **重启后端：** 修改 `characters_resource.json` 或 `scenes_resource.json` 后，需要重启后端服务才能生效（前端刷新页面即可看到变化）。
3. **坐标文件即时生效：** 修改 `positions_sample.json` 后，下次点击"下载坐标文件"就能获取最新版本，无需重启。
